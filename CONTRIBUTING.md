# Contributing

**TrustVault does not accept external contributions.**

This repository is published as **source-available** under the
[Business Source License 1.1 (BUSL-1.1)](LICENSE) so that anyone can read, study, audit, and — within the
terms of that license — test the code locally for non-production purposes. However, the project is
developed and maintained solely by its author, and the canonical repository is **not open to outside
contributions**.

Concretely, that means:

- ❌ **Pull requests will not be reviewed or merged** — they will be closed.
- ❌ **Direct commits / push access** are not granted to anyone.
- ❌ **Feature requests and code-change issues** are not accepted.
- ❌ There is **no contributor onboarding, CLA, or roadmap input** process.

Please do not open PRs or issues asking to contribute code — they will be declined without review.

## What you *can* do

Within the terms of the [Business Source License 1.1](LICENSE), you are welcome to:

- **Read, study, and audit** the code.
- **Copy and modify** it, and **make non-production use** of it (e.g. run it locally to test or evaluate).
- **Report a security vulnerability** privately — do **not** open a public issue for security problems;
  contact the author directly and privately instead.

What the license does **not** grant by default:

- ⛔ **Production or commercial use.** The Additional Use Grant is *None*. To use TrustVault in production
  or commercially before the Change Date, you must obtain a license or a targeted exception from the
  Licensor.
- The restrictions **automatically lift on the Change Date (2030-07-02)**, when the code converts to the
  Apache License 2.0.
